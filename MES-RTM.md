### MES-RTM前端页面需求规格说明

#### 文档说明：

| 文档属性 | 详情                                                         |
| -------- | ------------------------------------------------------------ |
| 适用系统 | SMT-MES 实时制造执行子系统（MES-RTM）                        |
| 技术架构 | Vue Web B/S 架构                                             |
| 适配终端 | 办公 PC、车间工位平板、车间大屏看板                          |
| 数据依赖 | 只读继承 MES-MDM 基础数据管理子系统的静态主数据，仅写入动态业务数 |



#### 一、前置规则说明

**权限规则**：前端仅做权限渲染与操作拦截，权限配置统一在 MES-MDM 子系统完成；核心适配角色：操作工、班组长、生产主管、质量工程师、工厂管理层，所有页面按角色权限做数据范围与操作按钮控制。

**实时性要求**：生产数据、设备状态、质量数据默认 30s 自动刷新，异常告警、批次锁定等关键事件实时推送。

**数据边界**：前端仅可读取 MDM 维护的静态主数据（设备、产品、BOM、工艺路线、参数模板等），不可编辑修改，所有静态数据修改入口均在 MDM 子系统。

**交互规范**：车间端页面优先适配扫码枪操作，核心操作按钮放大，减少手动输入项，保证操作工可快速完成核心操作。



#### 二、首页

**访问角色**：全角色（默认登录首页，按角色权限过滤数据范围）

**页面入口**：一级导航「首页」



##### 核心页面内容

1. 顶部核心指标卡片：当日工单总数、在制批次数量、整体直通率、当日计划完成率
2. 产线状态总览：按产线维度展示当前工单、生产进度、设备状态（运行 / 待机 / 故障 / 离线，绿 / 黄 / 红 / 灰标识）
3. 异常告警专区：实时展示未处理的批次锁定（跳转拦截管理页）、设备超差（跳转设备监控页）、质量异常（跳转拦截管理页）、物料缺料（跳转上料校验页）告警
4. 当日生产进度：工单维度的计划数量 / 已完工数量 / 完成率排行
5. 质量趋势图：当日 SPI/AOI 直通率、批次良率小时级趋势曲线

##### 核心操作

1. 点击产线卡片跳转至对应产线详情看板
2. 点击告警条目跳转至对应异常处理页面
3. 按车间、产线、日期范围筛选数据
4. 关键指标卡片下钻查看明细数据

##### 权限控制

- 管理层 / 生产主管：查看全车间所有产线数据
- 班组长：仅查看负责产线的数据
- 操作工 / 质量工程师：仅查看权限范围内的关联数据

##### 交互要点

默认展示当日实时数据，关键异常高亮置顶，支持大屏全屏展示







#### 三、生产调度模块

##### 3.1 工单管理列表页

- **访问角色**：生产主管、班组长
- **页面入口**：一级导航「生产调度」→ 二级导航「工单管理」

###### 核心页面内容

1. 高级筛选区：按工单号、产品型号、工单状态、交货期范围、分配产线筛选
2. 工单状态统计卡片：各状态工单数量统计
3. 工单列表表格，核心字段如下：

| 字段分类 | 核心字段                                                     |
| -------- | ------------------------------------------------------------ |
| 基础信息 | 工单号（规则：日期 + 流水号）、产品型号、产品名称、计划数量、交货期、分配产线 |
| 进度信息 | 已完工数量、完成率、已释放批次数量、已完成批次数量           |
| 状态信息 | 工单状态（待创建 / 待释放 / 生产中 / 暂停 / 已完成 / 已关闭）、创建人、创建时间、释放时间 |

###### 核心操作

1. 新建工单：弹窗填写工单基础信息，选择产品型号、分配产线、填写计划数量 / 交货期，提交生成工单
2. 工单释放：选中待释放工单，确认产线与批次拆分规则，释放工单至对应产线，自动生成生产批次
3. 工单状态变更：对生产中工单执行暂停、恢复操作；对已完工工单执行关闭操作
4. 详情查看：点击工单号跳转至工单详情页
5. 数据导出：工单列表数据 Excel 导出

###### 权限控制

- 生产主管：全量工单的新建、释放、暂停、关闭、查看权限
- 班组长：仅负责产线工单的查看权限，无编辑操作权限

###### 交互要点

已关闭工单不可编辑，仅可查看；释放工单时自动校验产品对应的工艺路线、参数模板是否完整，缺失则拦截并提示





##### 3.2 工单详情页

- **访问角色**：生产主管、班组长
- **页面入口**：工单管理列表页点击工单号进入

###### 核心页面内容

1. 工单基础信息卡片：工单号、产品信息、计划数量、交货期、分配产线、创建 / 释放 / 关闭时间、工单状态
2. 产品 BOM 信息：只读展示该产品的物料清单（元件料号、位号、用量、封装类型、替代料规则）
3. 工艺路线信息：只读展示该产品对应的工艺路线、工序序列、对应设备、标准工时
4. 批次列表：该工单下所有批次的明细，包含批次号、批次状态、计划数量、已完工数量、当前工序、上线 / 下线时间
5. 工单进度跟踪：工单整体完成率、各工序完工进度条形图

###### 核心操作

1. 批次操作：针对该工单新建、删除批次
2. 跳转操作：点击批次号跳转至批次详情页
3. 工单状态变更：同列表页的暂停、恢复、关闭操作
4. 单据打印：工单生产任务单打印

###### 权限控制

同工单管理列表页，仅生产主管可执行批次新建 / 删除、工单状态变更操作





##### 3.3 批次管理列表页

- **访问角色**：生产主管、班组长、操作工、质量工程师
- **页面入口**：一级导航「生产调度」→ 二级导航「批次管理」

###### 核心页面内容

1. 高级筛选区：按批次号、工单号、产品型号、批次状态、产线、日期范围筛选
2. 批次状态统计卡片：各状态批次数量统计
3. 批次列表表格，核心字段如下：

| 字段分类 | 核心字段                                                     |
| -------- | ------------------------------------------------------------ |
| 基础信息 | 批次号、所属工单号、产品型号、分配产线、计划数量             |
| 进度信息 | 已完工数量、不良数量、当前所在工序、预计完成时间、上线时间   |
| 状态信息 | 批次状态（待进站 / 生产中 / 暂停 / 维修中 / 已完成 / 已锁定）、锁定原因、负责人 |

###### 核心操作

1. 批次锁定 / 解锁：针对异常批次执行手动锁定 / 解锁操作，需填写锁定 / 解锁原因
2. 批次暂停 / 恢复：针对待料、设备故障等场景执行批次暂停 / 恢复操作，填写原因
3. 详情查看：点击批次号跳转至批次详情页
4. 批量操作：同产线批次的批量暂停 / 恢复操作

###### 权限控制

- 生产主管 / 班组长：全量批次操作权限
- 质量工程师：批次锁定 / 解锁、查看权限
- 操作工：仅负责产线的批次查看权限，无编辑操作权限

###### 交互要点

已锁定、维修中批次高亮显示；系统自动锁定的批次不可手动解锁，需质量工程师完成异常处置后解锁





##### 3.4 批次详情页

- **访问角色**：全角色
- **页面入口**：批次管理列表页点击批次号进入

###### 核心页面内容

1. 批次基础信息卡片：批次号、所属工单、产品信息、分配产线、计划数量、已完工 / 不良 / 报废数量、批次状态、上线 / 预计下线时间
2. 工序流转跟踪：时间轴形式展示该批次所有工序的进站 / 出站时间、完工数量、不良数量、操作人、设备信息
3. 上料记录：该批次所有工序的上料明细，包含站位号、物料料号、规格、上料数量、操作人、操作时间、校验结果
4. 质量检测记录：该批次 SPI/AOI 检测明细、直通率、不良项明细、异常处置记录
5. 工艺参数记录：该批次各工序使用的工艺参数模板、实际下发参数、设备校验结果

###### 核心操作

1. 流转操作：跳转至该批次当前工序的进站 / 出站操作页
2. 维修操作：跳转至该批次的维修操作页
4. 单据打印：批次流程卡打印

###### 权限控制

仅对应角色可执行进站 / 出站 / 维修操作，其余角色仅可查看

#### 四、生产执行模块

##### 4.1 进站操作页

- **访问角色**：操作工、班组长
- **页面入口**：一级导航「生产执行」→ 二级导航「进站操作」；批次详情页「进站操作」按钮

###### 核心页面内容

1. 批次选择区：扫码输入批次号 / 下拉选择当前产线待进站批次，展示批次基础信息（产品型号、计划数量、当前工序、上一工序出站情况）

2. 进站校验区：

   - 工序合规性校验结果：上一工序是否完成出站、是否允许跳工序
   - 设备选择：下拉选择当前工序对应设备，仅展示该产线该工序类型的可用设备
   - 上料校验结果：当前批次该工序所需物料的上料完成情况、校验通过率

   

3. 进站信息填写区：进站数量（默认批次剩余数量，可修改）、操作人、备注

###### 核心操作

1. 扫码识别：支持扫描批次条码、设备条码快速填充信息
2. 上料操作：跳转至「上料校验操作页」完成该批次的上料与校验
3. 进站提交：校验全部通过后，提交完成进站操作，系统记录进站时间、上料信息、操作人
4. 信息重置：清空当前填写的进站信息

###### 权限控制

仅当前产线的操作工 / 班组长可执行进站操作

###### 交互要点

- 校验不通过时，明确展示错误原因（未选择设备、物料未完成上料、上一工序未出站等），并拦截进站提交
- 操作界面简化，适配车间平板扫码操作，核心按钮放大，减少输入项
- 进站成功后自动跳转至对应批次的出站操作页





##### 4.2 上料校验操作页

- **访问角色**：操作工、班组长
- **页面入口**：进站操作页「上料操作」按钮；一级导航「生产执行」→ 二级导航「上料校验」

###### 核心页面内容

1. 批次与工序信息：只读展示当前批次、产品型号、当前工序、对应设备
2. 上料进度条：当前工序已完成上料的站位占比
3. 扫码操作区：物料条码输入框、站位码输入框、校验结果实时提示
4. 上料任务清单表格，核心字段：站位号、物料料号、物料规格、封装类型、应上数量、已上数量、校验状态、替代料信息

###### 核心操作

1. 扫码校验：操作员依次扫描物料条码→扫描站位码，系统自动校验物料与 BOM 清单的一致性（料号、封装、替代料规则），校验通过自动标记该站位上料完成
2. 替代料确认：扫描替代料时，弹窗展示替代料优先级与规则，确认后完成上料
3. 批量校验：批量导入上料清单，执行批量校验
4. 记录查看：查看该站位历史上料记录

###### 权限控制

仅当前产线的操作工 / 班组长可执行上料操作

###### 交互要点

- 校验失败时，弹窗高亮提示错误原因（料号不匹配、封装不匹配、无替代料权限），并拦截上料确认
- 已完成上料的站位标绿，未完成标灰，校验失败标红

##### 4.3 出站操作页

- **访问角色**：操作工、班组长
- **页面入口**：一级导航「生产执行」→ 二级导航「出站操作」；进站成功后自动跳转；批次详情页「出站操作」按钮

###### 核心页面内容

1. 批次与工序基础信息：只读展示批次号、产品型号、当前工序、进站时间、进站数量、对应设备

2. 出站数量填写区：

   - 完工数量：默认进站数量，可修改
   - 不良数量：填写后自动带出不良处置选项
   - 报废数量：填写后需填写报废原因

   

3. 不良处置区：不良数量＞0 时展示，可选处置方式：维修、报废、强制出站，强制出站需填写强制原因

4. 质量检测数据区：按工序类型差异化展示

   - 自动化工序：自动拉取设备检测数据，减少手动输入
     - SPI 检测：锡膏体积/面积/高度合格率、偏移超差数、直通率
     - 贴片工序：贴装数量、抛料数量、抛料率、吸嘴真空度、供料器状态
     - 回流焊接：各温区实际温度与设定值偏差、链速、炉后检测直通率
     - AOI 检测：不良项明细（缺件/偏移/立碑/桥接/极性反）、直通率
   - 人工工序：支持手动录入检测结果，填写检测项与判定结论

5. 操作人信息、备注填写区

###### 核心操作

1. 出站提交：确认信息无误后提交，系统记录出站时间、完工 / 不良 / 报废数量、操作人，完成工序流转
2. 不良提报：不良数量＞0 时，提交后自动生成不良记录，需维修的批次自动流转至维修管理页
3. 单据打印：出站流程单打印

###### 权限控制

- 操作工 / 班组长：正常出站、维修 / 报废处置操作权限
- 生产主管：强制出站操作权限，普通操作工无此权限

###### 交互要点

- 区分自动化工序与人工工序的界面：自动化工序以数据确认、决策操作为主，减少手动输入；人工工序以数据录入、结果记录为主
- 核心操作按钮放大，适配车间平板操作，保证操作工 30 秒内完成正常出站操作
- 批次被质量系统锁定时，拦截出站操作，提示锁定原因与处置流程





##### 4.4 维修管理页

- **访问角色**：维修员、班组长、质量工程师
- **页面入口**：一级导航「生产执行」→ 二级导航「维修管理」；出站操作页不良处置跳转

###### 核心页面内容

1. 维修概览卡片：待维修批次总数、今日已维修批次、维修合格率、待维修超期预警

2. 待维修批次列表：核心字段包含批次号、产品型号、所属产线、不良工序、不良数量、不良项明细、提报时间、超期标识

3. 维修操作区：选中批次后展开，包含：

   - 批次与不良基础信息
   - 维修记录填写区：维修方法、维修结果、维修数量、报废数量、维修人、维修时间
   - 不良原因分析、预防措施填写区

   

4. 已维修批次历史记录列表
