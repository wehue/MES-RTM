# MES-RTM 系统需求规格与操作逻辑整合文档


## 1. 用户角色与权限基础说明

用户说明：

用户按照老师的需求是只能在MDM系统中的用户管理进行新增，在两个系统中不提供注册功能

角色表说明:

角色是和子系统绑定，比如工艺工程师只能登录MDM子系统，无法登录RTM系统

功能表说明:

功能只控制到模块，比如MDM系统中只控制到工艺管理模块(而不控制这个模块下的新增工艺路线这些详细)，注:RTM系统的功能模块可以考虑按照前端的左侧进行划分

用户和角色之间的关系:一个用户可以拥有多个角色(如一个用户可以同时有生产主管和管理人员两个角色)，一个角色可以被分配给多个用户

角色和功能的关系:一个角色可以被分配相应的功能模块

用户角色功能的关系：这里的角色其实就是权限的意思，一个用户有怎样的角色，他就拥有相应的角色功能，就相当于一个用户有怎么权限，他就有相应的权限功能

## 2. MES-RTM 前端页面需求规格说明

### MES-RTM前端页面需求规格说明

#### 文档说明：


| 文档属性 | 详情                                    |
| ---- | ------------------------------------- |
| 适用系统 | SMT-MES 实时制造执行子系统（MES-RTM）            |
| 技术架构 | Vue Web B/S 架构                        |
| 适配终端 | 办公 PC、车间工位平板、车间大屏看板                   |
| 数据依赖 | 只读继承 MES-MDM 基础数据管理子系统的静态主数据，仅写入动态业务数 |


#### 一、前置规则说明

**权限规则**：前端仅做权限渲染与操作拦截，权限配置统一在 MES-MDM 子系统完成；核心适配角色：操作工、班组长、生产主管、质量工程师、工厂管理层，所有页面按角色权限做数据范围与操作按钮控制。

**实时性要求**：生产数据、设备状态、质量数据默认 30s 自动刷新，异常告警、批次锁定等关键事件实时推送。

**数据边界**：前端仅可读取 MDM 维护的静态主数据（设备、产品、BOM、工艺路线、参数模板等），不可编辑修改，所有静态数据修改入口均在 MDM 子系统。

**交互规范**：车间端页面优先适配扫码枪操作，核心操作按钮放大，减少手动输入项，保证操作工可快速完成核心操作。

#### 二、首页

**访问角色**：全角色（默认登录首页，按角色权限过滤数据范围）

**页面入口**：一级导航「首页」

##### 核心页面内容

1. 顶部核心指标卡片：当日工单总数、在制批次数量、整体直通率、当日计划完成率
2. 产线状态总览：按产线维度展示当前工单、生产进度、设备状态（运行 / 待机 / 故障 / 离线，绿 / 黄 / 红 / 灰标识）
3. 异常告警专区：实时展示未处理的批次锁定（跳转拦截管理页）、设备超差（跳转设备监控页）、质量异常（跳转拦截管理页）、物料缺料（跳转上料校验页）告警
4. 当日生产进度：工单维度的计划数量 / 已完工数量 / 完成率排行
5. 质量趋势图：当日 SPI/AOI 直通率、批次良率小时级趋势曲线

##### 核心操作

1. 点击产线卡片跳转至对应产线详情看板
2. 点击告警条目跳转至对应异常处理页面
3. 按车间、产线、日期范围筛选数据
4. 关键指标卡片下钻查看明细数据

##### 权限控制

- 管理层 / 生产主管：查看全车间所有产线数据
- 班组长：仅查看负责产线的数据
- 操作工 / 质量工程师：仅查看权限范围内的关联数据

##### 交互要点

默认展示当日实时数据，关键异常高亮置顶，支持大屏全屏展示

#### 三、生产调度模块

##### 3.1 工单管理列表页

- **访问角色**：生产主管、班组长
- **页面入口**：一级导航「生产调度」→ 二级导航「工单管理」

###### 核心页面内容

1. 高级筛选区：按工单号、产品型号、工单状态、交货期范围、分配产线筛选
2. 工单状态统计卡片：各状态工单数量统计
3. 工单列表表格，核心字段如下：


| 字段分类 | 核心字段                                                 |
| ---- | ---------------------------------------------------- |
| 基础信息 | 工单号（规则：日期 + 流水号）、产品型号、产品名称、计划数量、交货期、分配产线             |
| 进度信息 | 已完工数量、完成率、已释放批次数量、已完成批次数量                            |
| 状态信息 | 工单状态（待创建 / 待释放 / 生产中 / 暂停 / 已完成 / 已关闭）、创建人、创建时间、释放时间 |


###### 核心操作

1. 新建工单：弹窗填写工单基础信息，选择产品型号、分配产线、填写计划数量 / 交货期，提交生成工单
2. 工单释放：选中待释放工单，确认产线与批次拆分规则，释放工单至对应产线，自动生成生产批次
3. 工单状态变更：对生产中工单执行暂停、恢复操作；对已完工工单执行关闭操作
4. 详情查看：点击工单号跳转至工单详情页
5. 数据导出：工单列表数据 Excel 导出

###### 权限控制

- 生产主管：全量工单的新建、释放、暂停、关闭、查看权限
- 班组长：仅负责产线工单的查看权限，无编辑操作权限

###### 交互要点

已关闭工单不可编辑，仅可查看；释放工单时自动校验产品对应的工艺路线、参数模板是否完整，缺失则拦截并提示

##### 3.2 工单详情页

- **访问角色**：生产主管、班组长
- **页面入口**：工单管理列表页点击工单号进入

###### 核心页面内容

1. 工单基础信息卡片：工单号、产品信息、计划数量、交货期、分配产线、创建 / 释放 / 关闭时间、工单状态
2. 产品 BOM 信息：只读展示该产品的物料清单（元件料号、位号、用量、封装类型、替代料规则）
3. 工艺路线信息：只读展示该产品对应的工艺路线、工序序列、对应设备、标准工时
4. 批次列表：该工单下所有批次的明细，包含批次号、批次状态、计划数量、已完工数量、当前工序、上线 / 下线时间
5. 工单进度跟踪：工单整体完成率、各工序完工进度条形图

###### 核心操作

1. 批次操作：针对该工单新建、删除批次
2. 跳转操作：点击批次号跳转至批次详情页
3. 工单状态变更：同列表页的暂停、恢复、关闭操作
4. 单据打印：工单生产任务单打印

###### 权限控制

同工单管理列表页，仅生产主管可执行批次新建 / 删除、工单状态变更操作

##### 3.3 批次管理列表页

- **访问角色**：生产主管、班组长、操作工、质量工程师
- **页面入口**：一级导航「生产调度」→ 二级导航「批次管理」

###### 核心页面内容

1. 高级筛选区：按批次号、工单号、产品型号、批次状态、产线、日期范围筛选
2. 批次状态统计卡片：各状态批次数量统计
3. 批次列表表格，核心字段如下：


| 字段分类 | 核心字段                                            |
| ---- | ----------------------------------------------- |
| 基础信息 | 批次号、所属工单号、产品型号、分配产线、计划数量                        |
| 进度信息 | 已完工数量、不良数量、当前所在工序、预计完成时间、上线时间                   |
| 状态信息 | 批次状态（待进站 / 生产中 / 暂停 / 维修中 / 已完成 / 已锁定）、锁定原因、负责人 |


###### 核心操作

1. 批次锁定 / 解锁：针对异常批次执行手动锁定 / 解锁操作，需填写锁定 / 解锁原因
2. 批次暂停 / 恢复：针对待料、设备故障等场景执行批次暂停 / 恢复操作，填写原因
3. 详情查看：点击批次号跳转至批次详情页
4. 批量操作：同产线批次的批量暂停 / 恢复操作

###### 权限控制

- 生产主管 / 班组长：全量批次操作权限
- 质量工程师：批次锁定 / 解锁、查看权限
- 操作工：仅负责产线的批次查看权限，无编辑操作权限

###### 交互要点

已锁定、维修中批次高亮显示；系统自动锁定的批次不可手动解锁，需质量工程师完成异常处置后解锁

##### 3.4 批次详情页

- **访问角色**：全角色
- **页面入口**：批次管理列表页点击批次号进入

###### 核心页面内容

1. 批次基础信息卡片：批次号、所属工单、产品信息、分配产线、计划数量、已完工 / 不良 / 报废数量、批次状态、上线 / 预计下线时间
2. 工序流转跟踪：时间轴形式展示该批次所有工序的进站 / 出站时间、完工数量、不良数量、操作人、设备信息
3. 上料记录：该批次所有工序的上料明细，包含站位号、物料料号、规格、上料数量、操作人、操作时间、校验结果
4. 质量检测记录：该批次 SPI/AOI 检测明细、直通率、不良项明细、异常处置记录
5. 工艺参数记录：该批次各工序使用的工艺参数模板、实际下发参数、设备校验结果

###### 核心操作

1. 流转操作：跳转至该批次当前工序的进站 / 出站操作页
2. 维修操作：跳转至该批次的维修操作页
3. 单据打印：批次流程卡打印

###### 权限控制

仅对应角色可执行进站 / 出站 / 维修操作，其余角色仅可查看

#### 四、生产执行模块

##### 4.1 进站操作页

- **访问角色**：操作工、班组长
- **页面入口**：一级导航「生产执行」→ 二级导航「进站操作」；批次详情页「进站操作」按钮

###### 核心页面内容

1. 批次选择区：扫码输入批次号 / 下拉选择当前产线待进站批次，展示批次基础信息（产品型号、计划数量、当前工序、上一工序出站情况）
2. 进站校验区：
  - 工序合规性校验结果：上一工序是否完成出站、是否允许跳工序
  - 设备选择：下拉选择当前工序对应设备，仅展示该产线该工序类型的可用设备
  - 上料校验结果：当前批次该工序所需物料的上料完成情况、校验通过率
3. 进站信息填写区：进站数量（默认批次剩余数量，可修改）、操作人、备注

###### 核心操作

1. 扫码识别：支持扫描批次条码、设备条码快速填充信息
2. 上料操作：跳转至「上料校验操作页」完成该批次的上料与校验
3. 进站提交：校验全部通过后，提交完成进站操作，系统记录进站时间、上料信息、操作人
4. 信息重置：清空当前填写的进站信息

###### 权限控制

仅当前产线的操作工 / 班组长可执行进站操作

###### 交互要点

- 校验不通过时，明确展示错误原因（未选择设备、物料未完成上料、上一工序未出站等），并拦截进站提交
- 操作界面简化，适配车间平板扫码操作，核心按钮放大，减少输入项
- 进站成功后自动跳转至对应批次的出站操作页

##### 4.2 上料校验操作页

- **访问角色**：操作工、班组长
- **页面入口**：进站操作页「上料操作」按钮；一级导航「生产执行」→ 二级导航「上料校验」

###### 核心页面内容

1. 批次与工序信息：只读展示当前批次、产品型号、当前工序、对应设备
2. 上料进度条：当前工序已完成上料的站位占比
3. 扫码操作区：物料条码输入框、站位码输入框、校验结果实时提示
4. 上料任务清单表格，核心字段：站位号、物料料号、物料规格、封装类型、应上数量、已上数量、校验状态、替代料信息

###### 核心操作

1. 扫码校验：操作员依次扫描物料条码→扫描站位码，系统自动校验物料与 BOM 清单的一致性（料号、封装、替代料规则），校验通过自动标记该站位上料完成
2. 替代料确认：扫描替代料时，弹窗展示替代料优先级与规则，确认后完成上料
3. 批量校验：批量导入上料清单，执行批量校验
4. 记录查看：查看该站位历史上料记录

###### 权限控制

仅当前产线的操作工 / 班组长可执行上料操作

###### 交互要点

- 校验失败时，弹窗高亮提示错误原因（料号不匹配、封装不匹配、无替代料权限），并拦截上料确认
- 已完成上料的站位标绿，未完成标灰，校验失败标红

##### 4.3 出站操作页

- **访问角色**：操作工、班组长
- **页面入口**：一级导航「生产执行」→ 二级导航「出站操作」；进站成功后自动跳转；批次详情页「出站操作」按钮

###### 核心页面内容

1. 批次与工序基础信息：只读展示批次号、产品型号、当前工序、进站时间、进站数量、对应设备
2. 出站数量填写区：
  - 完工数量：默认进站数量，可修改
  - 不良数量：填写后自动带出不良处置选项
  - 报废数量：填写后需填写报废原因
3. 不良处置区：不良数量＞0 时展示，可选处置方式：维修、报废、强制出站，强制出站需填写强制原因
4. 质量检测数据区：按工序类型差异化展示
  - 自动化工序：自动拉取设备检测数据，减少手动输入
    - SPI 检测：锡膏体积/面积/高度合格率、偏移超差数、直通率
    - 贴片工序：贴装数量、抛料数量、抛料率、吸嘴真空度、供料器状态
    - 回流焊接：各温区实际温度与设定值偏差、链速、炉后检测直通率
    - AOI 检测：不良项明细（缺件/偏移/立碑/桥接/极性反）、直通率
  - 人工工序：支持手动录入检测结果，填写检测项与判定结论
5. 操作人信息、备注填写区

###### 核心操作

1. 出站提交：确认信息无误后提交，系统记录出站时间、完工 / 不良 / 报废数量、操作人，完成工序流转
2. 不良提报：不良数量＞0 时，提交后自动生成不良记录，需维修的批次自动流转至维修管理页
3. 单据打印：出站流程单打印

###### 权限控制

- 操作工 / 班组长：正常出站、维修 / 报废处置操作权限
- 生产主管：强制出站操作权限，普通操作工无此权限

###### 交互要点

- 区分自动化工序与人工工序的界面：自动化工序以数据确认、决策操作为主，减少手动输入；人工工序以数据录入、结果记录为主
- 核心操作按钮放大，适配车间平板操作，保证操作工 30 秒内完成正常出站操作

## 3. MES-RTM 整体操作逻辑

**整体操作逻辑**
这个系统可以按“计划下达 -> 批次释放 -> 现场执行 -> 质量拦截 -> 维修解锁 -> 看板监控”的 MES 闭环来操作。

**1. 登录与角色选择**

```
账号：admin 密码：123456 
```

然后选择演示角色。不同角色影响页面按钮权限：

- 工厂管理层：偏查看全局数据、看板、驾驶舱。

- 生产主管：可新建工单、释放工单、暂停/关闭工单、强制出站、查看全局生产。

- 班组长：可查看负责产线，可执行部分批次、进站、上料、出站、维修协同操作。

- 操作工：主要执行进站、上料、出站、批次查看。

- 质量工程师：主要处理阈值配置、质量拦截、批次锁定/解锁审核。

- 维修员：主要处理维修接单、维修记录、维修提交。

   

**2. 首页生产驾驶舱**
登录后默认进入“首页 / 生产驾驶舱”。

这里用于管理层、生产主管、班组长快速判断现场状态：

- 顶部看当日工单总数、在制批次数、整体直通率、计划完成率。
- 中间看每条产线当前工单、产品型号、进度、OEE、设备状态。
- 右侧看异常告警，比如批次锁定、设备故障、物料缺料、质量异常。
- 点击产线卡片会进入产线状态看板。
- 点击告警条目会跳转到对应处理页面，比如质量拦截、设备监控、上料校验。

这一页的逻辑是“先看异常，再看产线，再看工单进度和质量趋势”。



**3. 工单管理流程**
路径：生产调度 -> 工单管理

生产主管的标准操作步骤：

1. 点击“新建工单”。
2. 选择产品型号。
3. 选择分配产线。
4. 输入计划数量。
5. 选择交货期。
6. 提交生成工单。
7. 在工单列表中找到“待释放”工单。
8. 点击“释放”。
9. 系统模拟校验 BOM、工艺路线、参数模板是否完整。
10. 校验通过后，工单释放到产线，并生成生产批次。
11. 生产过程中可以对“生产中”工单执行暂停。
12. 已完成工单可以执行关闭。
13. 点击工单号进入工单详情，查看 BOM、工艺路线、批次列表、工序进度。

班组长进入此页面时，主要是查看工单状态和详情，编辑类操作会被禁用或拦截。



**4. 批次管理流程**
路径：生产调度 -> 批次管理

这里是工单释放后的核心批次管控页面。

标准操作步骤：

1. 按批次号、工单号、产品型号、状态、产线筛选批次。
2. 查看批次状态卡片，判断待进站、生产中、暂停、维修中、已完成、已锁定数量。
3. 对异常批次执行“锁定”，需要填写原因。
4. 对已锁定批次执行“解锁”，系统会判断是否为自动锁定。
5. 如果是系统自动锁定，比如直通率低于阈值，则不能手动解锁，必须走质量拦截处理。
6. 对待料、设备故障场景，可以执行暂停。
7. 异常解除后执行恢复。
8. 点击批次号进入批次详情。

批次详情里可以查看：

- 批次基础信息。

- 工序流转时间轴。

- 上料记录。

- 质量检测记录。

- 工艺参数记录。

- 并可跳转进站、出站、维修操作页。

   

**5. 进站操作流程**
路径：生产执行 -> 进站操作

这是操作工或班组长的现场操作页面，适合扫码枪和平板。

标准操作步骤：

1. 扫描或选择批次号。
2. 系统展示批次产品型号、计划数量、当前工序、上一工序出站情况。
3. 系统进行工序合规校验。
4. 扫描或选择当前工序可用设备。
5. 查看上料校验状态。
6. 如果上料未完成，点击“上料操作”跳转到上料校验。
7. 输入或确认进站数量。
8. 确认操作人和备注。
9. 点击“提交进站”。
10. 校验全部通过后，系统记录进站时间、设备、上料信息、操作人。
11. 成功后跳转到出站操作页。

拦截逻辑：

- 未选择设备，不能进站。

- 批次锁定，不能进站。

- 上料未完成时会提示去补齐上料校验。

   

**6. 上料校验流程**
路径：生产执行 -> 上料校验

标准操作步骤：

1. 查看当前批次、产品型号、当前工序、对应设备。
2. 查看上料完成率。
3. 扫描物料条码。
4. 扫描站位码。
5. 点击“扫码校验”。
6. 系统判断站位是否存在。
7. 系统判断物料料号是否与 BOM 一致。
8. 校验通过后，该站位自动标记为“已完成”。
9. 校验失败时，弹窗提示原因，比如料号不匹配、站位不存在、替代料无权限。
10. 所有关键站位完成后，返回进站操作继续提交。

页面中的颜色逻辑：

- 已完成：绿色。

- 待完成：黄色。

- 失败：红色。

   

**7. 出站操作流程**
路径：生产执行 -> 出站操作

标准操作步骤：

1. 查看批次号、产品型号、当前工序、进站时间、对应设备。
2. 填写或确认完工数量。
3. 填写不良数量。
4. 填写报废数量。
5. 如果不良数量大于 0，选择不良处置方式：
  - 维修
  - 报废
  - 强制出站
6. 如果选择强制出站，只有生产主管角色允许提交。
7. 查看自动采集的质量检测数据，比如 SPI、AOI、批次良率。
8. 填写操作人和备注。
9. 点击“提交出站”。
10. 系统记录出站时间、完工、不良、报废、质量数据和操作人。
11. 如果有不良且选择维修，会流转到维修管理。

拦截逻辑：

- 批次被质量系统锁定时，禁止出站。

- 普通操作工不能强制出站。

- 锁定原因会直接显示给操作人员。

   

**8. 维修管理流程**
路径：生产执行 -> 维修管理

标准操作步骤：

1. 查看待维修批次、今日已维修、维修合格率、超期预警。

2. 在待维修批次列表选择一个批次。

3. 点击“维修接单”。

4. 填写维修方法。

5. 选择维修结果。

6. 填写维修数量、报废数量。

7. 填写不良原因分析。

8. 填写预防措施。

9. 点击“维修提交”。

10. 如果维修完成后需要解锁，由质量工程师点击“审核解锁”。

11. 审核通过后，批次恢复生产流转。

    

**9. 批次跟踪流程**
路径：生产执行 -> 批次跟踪

这个页面用于追溯。

标准操作步骤：

1. 扫描或输入批次号。
2. 点击查询。
3. 查看完整工序时间轴。
4. 查看每道工序的进站、出站、设备、操作人、完工数量、不良数量。
5. 查看上料记录汇总。
6. 查看操作日志。
7. 点击“导出追溯报告”生成追溯数据。

适用场景：

- 客诉追溯。

- 质量复盘。

- 批次异常定位。

- 工艺参数与质量数据核对。

   

**10. 设备监控流程**
路径：生产执行 -> 设备监控

标准操作步骤：

1. 查看设备总数、运行、待机、故障、离线数量。
2. 按设备类型筛选，比如印刷机、SPI、贴片机、回流炉、AOI。
3. 按设备状态筛选，比如运行、待机、故障、离线。
4. 按产线筛选。
5. 切换列表视图或矩阵视图。
6. 故障设备会红色高亮。
7. OEE 低于目标值的设备会黄色预警。
8. 点击“故障提报”提交故障工单。
9. 点击“保养确认”更新设备保养状态。
10. 下方查看故障设备详情和保养提醒。



**11. 看板中心操作**
路径：

- 看板中心 -> 产线状态看板
- 看板中心 -> 质量监控看板
- 看板中心 -> 生产进度看板

产线状态看板用于车间大屏：

2. 查看每条产线当前工单、产品型号、计划/完工、预计完成时间。
3. 查看设备状态色块。
4. 查看异常告警滚动区。
5. 查看当日工单进度排行。

质量监控看板用于质量部大屏：

1. 查看 SPI 直通率。

3. 查看批次良率。
4. 查看不良率。
5. 查看拦截批次数。
6. 查看直通率趋势、不良类型分布、产线质量排行、拦截明细。

生产进度看板用于生产主管和管理层：

1. 查看 7 天内交货工单。
2. 查看未完成和延期预警。
3. 查看计划与实际对比甘特图。

5. 查看产线当日/月度产能达成率。

**12. 个人中心与消息中心**
个人中心：

## 4. 批次工序数据特征补充说明

**一、 批次状态（整体生命周期）**

这是批次的主状态，存储在批次主表中，用于宏观管理和看板展示。


| 状态名称    | 英文               | 含义                                           | 触发动作/条件                                 |
| ------- | ---------------- | -------------------------------------------- | --------------------------------------- |
| **待生产** | Pending          | 批次已创建，计划已下达，但尚未开始任何生产活动。                     | 工单释放后，批次拆分生成。                           |
| **生产中** | Running          | 批次投产，已进入第一道工序且尚未完成最后一道工序。                    | 第一个工序完成“进站”操作后，由 Pending 转为 Running。    |
| **暂停**  | On Hold / Paused | 批次因异常情况（如缺料、设备故障、质量问题待评审）被临时冻结，不能继续流转。       | 班组长或系统（如质量锁定）在任意工序执行“暂停”操作。             |
| **维修中** | Rework           | 批次因质量检测不合格，被送入维修站进行维修。这是一个特殊的生产中状态，但通常会单独标识。 | AOI检测不合格后，批次被转入维修路径。                    |
| **已锁定** | Locked           | 批次因严重的质量或安全问题被强制锁定，通常需要高级别审批才能解锁。            | 直通率远低于阈值、客户紧急叫停等触发系统自动或人工锁定。            |
| **已完成** | Completed        | 批次已通过了所有规定的工序，完成了全部生产任务。                     | 最后一个工序完成“出站”操作后，由 Running 转为 Completed。 |


**二、 工序状态（微观执行进度）**

这是批次的细节状态，用于指导现场操作和精确控制流转，存储在批次工序状态表中。


| 状态名称    | 英文                      | 含义                                | 操作/动作                          |
| ------- | ----------------------- | --------------------------------- | ------------------------------ |
| **待进站** | Pending / Wait-In       | 批次已完成上一道工序，在当前工序的缓存区排队，等待进入。      | 上一工序“出站”后，自动进入当前工序的“待进站”状态。    |
| **已进站** | Checked-In / In-Process | 批次已被操作工扫码接收，正式进入当前工序，但尚未完成加工。     | 操作工执行“进站”操作。系统记录进站时间和数量。       |
| **已出站** | Checked-Out / Completed | 批次已完成当前工序的所有加工和检测任务，准备或已流转至下一工序。  | 操作工执行“出站”操作。系统记录出站时间、产量和质量数据。  |
| **暂停**  | Paused / On Hold        | 因非质量因素（缺料、设备故障等）被冻结               | 班组长手动暂停                        |
| **锁定**  | Locked                  | 批次在当前工序被锁定，无法进行进站或出站操作，通常由质量异常触发。 | SPI/AOI检测直通率不达标，班组长手动锁定。       |
| **跳过**  | Skipped                 | 批次跳过当前工序。此状态只在特殊配置下使用。            | 工艺路线中设置了“允许跳过”，且操作工或系统执行了“跳过”。 |


**场景描述：批次 BATCH-002 准备进入“高速贴片”工序，但班组长发现该批次所需的某种物料（如100kΩ电阻）尚未完成上料，于是暂停批次，等待物料到位后再恢复生产。**

**完整状态流转示例：**


| **序号** | **事件**            | **批次状态** | **工序状态（当前工序）**      | **说明**         |
| ------ | ----------------- | -------- | ------------------- | -------------- |
| **1**  | **工单释放，批次生成**     | **待生产**  | **（无）**             | **批次刚创建**      |
| **2**  | **进入第一道工序（印刷）**   | **生产中**  | **待进站 → 已进站 → 已出站** | **正常流转**       |
